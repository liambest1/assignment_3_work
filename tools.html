Tools

Microsoft Teams

We have setup a Microsoft Teams group in which all our members are part of. Within Teams, we’ve setup a basic folder structure to store upcoming assessment contents as well as minutes and recordings to refer to should any of us miss a meeting.

Links to our Weekly Meetings can be found here which contain minutes recorded by the chair and recordings in a separate folder. Our meetings are held twice a week over Microsoft Teams on a regular schedule, every Tuesday between 6pm to 6:45pm and every Thursday between 8pm to 9pm. We have an agreed upon roster based on our Minutes Chair Process document so that everyone has a chance to experience hosting a meeting.

Microsoft SharePoint

Our main collaborative tool thought this group project has been SharePoint as this is where we have been storing all our work. We use one central SharePoint under Aileen's account that we all have access to. We use a single word doc that contains all our collaborated work, this has been effective because multiple people can work on the document at the same time and it will update in real time as we are all working in different locations and not physically together. SharePoint also has the advantage of being able to store all of our Meeting minutes and full recordings so that any team member that has missed a meeting can easily catch up in their own time. Below are screenshots of our version history.
Another advantage with using SharePoint is the version control function. It would allow us to revert back to a previous version should the document be either corrupted or missing segments at a certain point.

Monday.com

We used Monday.com as it’s a free platform that allows us to scale, assign and track progress of work thru various phases of different projects such as Assessment 3, Assessment 5 and the Quick Class Website. It has useful features in providing a Gnatt chart based on below table:



The table allows for team members to be assigned several tasks or sub items, update status (whether it has been completed, stuck, done, etc), priority and rough timeline that it is due. Based on the table, there is also a calendar view that provides further a useful birdseye overview on how well we’ve planned out tasks. From this view, we can move item(s) around on timeline if we see that we are overwhelmed or if there is a problem that is causing project to stall.
